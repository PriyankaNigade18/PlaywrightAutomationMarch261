api test

To Run playwright testcase 
-----------------------------
1.this command execute all test cases from tests folder
npx playwright test

2.To run specific test case
npx playwright test tests/Tc03_LaunchTestInMsEdge.spec.ts

3.To run test case in headed view(with browser/ui)
npx playwright test --headed

4.To run test case on one browser
npx playwright test --headed --project=chromium

5.UI Execution
npx playwright test --ui

Assertion
================
To validate current state of the application we use assertion
In Playwright two types of assertion supported
1.Hard Assertion
-----------------------
When we apply any assertion and if assertion get fail then execution 
will stop on that line and you will assertion error

we use expect() for assertion

2.Soft Assertion
-----------------------
When we apply any assertion and if assertion get fail then execution 
will not stop and assertion error you will after remaining test execution completed

we use expect.soft() for assertion

await expect(page).toHaveURL("full url");
await expect(page).toHaveURL(/partial url/);
await expect(page).toHaveTitle("full title");

goto()
==========
Page.goto(url: string, options?: {
    referer?: string | undefined;
    timeout?: number | undefined;
    waitUntil?: "load" | "domcontentloaded" | "networkidle" | "commit" | undefined;
} | undefined): Promise<Response | null>

'domcontentloaded' - consider operation to be finished when the DOMContentLoaded event is fired.
'load' - consider operation to be finished when the load event is fired.
'networkidle' - DISCOURAGED consider operation to be finished when there are no network connections for at least 500 ms. Don't use this method for testing, rely on web assertions to assess readiness instead.
'commit' - creceived anonsider operation to be finished when network response is d the document started loading.

How to automate WebElements in Playwright
===========================================

What is Webelement?
=====================
any ui control from the webpage 
for example textbox,link,text,heading,button,checkbox,radio,dropdown..etc

To automate webelement
-----------------------
1.Identify webelement from the webpage
2.perform action on those webelement

To identify webelement we use Locator

What is Locator?
--------------------
Locator is address of the webelement from webpage

Playwright support 3 Locator strategies
========================================
1.InBuilt Locators(7 InBuilt locator)
2.CssSelector
3.Xpath

Actions of webelement
======================
textbox,searchbox-->type--->fill()
button,link,dropdown--->click---->click()
checkbox,radiobutton-->click--->click(),check(),uncheck()
To clear the text: clear()

7 InBuilt Locator Methods
===============================
These are the recommended built-in locators.

page.getByRole() to locate by explicit and implicit accessibility attributes.
page.getByText() to locate by text content.
page.getByLabel() to locate a form control by associated label's text.
page.getByPlaceholder() to locate an input by placeholder.
page.getByAltText() to locate an element, usually image, by its text alternative.
page.getByTitle() to locate an element by its title attribute.
page.getByTestId() to locate an element based on its data-testid attribute (other attributes can be configured).

CssSelector
=======================
-Css is cascaded stylesheet this is language used to provide style to the webelement

-cssSelector is locator to identify webelement
-cssSelector identify element based on cssProperty
-cssSelector can travel in one direction parent to child and that is reason
cssselector is faster than xpath

1.tagname with id
2.tagname with attribute
3.tagname with classname
4.tagname with attribute,classname and id


The method returns an element locator that can be used to perform actions on this page / frame.
page.locator("cssSelector/xpath");


1.tagname with id
-----------------------
Syntax:
===========
tagname#id
OR
#id

<textarea id="APjFqb">
example:
textarea#APjFqb    1 of 1

2.tagname with attribute
-------------------------------
Syntax:
-------------
tagname[attribute='value']


3.tagname with className
-----------------------------
Syntax
----------
tagname.className

For class name In CssSelector
----------------------
<button class="btn btn-default btn-primary"/>
                c1      c2        c3

button.btn.btn-default.btn-primary 1 of 1
OR
button.btn    1 of 1


4.tagname with id,classname,attribute
----------------------------------------
Syntax
---------
tagname#id.className[attribute='value']

Xpath Locator
==============
-xpath is path of the element from xml/html document
-xpath identify element from DOM(Document object model)
-Xpath support methods
-xpath can travel in both direction like parent to child node or child to parent node
-xpath also support xpath axies(ancestor,parent,child,following,preceding,sibling)

html DOM
------------
<html>-----------------Root element
<head>
</head>

<body>
<input>
<a>----------element
<div>

</body>
</html>

===============================
Types of Xpath
=================
1.Absolute Xpath
--------------------
- Absolute xpath identify element from the root node
- starts with '/'

/html/body/div[2]/div/div/div/div[2]/div/form/div[1]/input

2.Relative Xpath
-----------------------
- Relative xpath identify element from elements relative property
- starts with '//'

//*[@id="input-email"]
here * identify any tag whose id is input-email

Syntax
===========
//tagname[@attribute='value']


========================================================
Xpath with operators
========================
1.and
Syntax
------------
//tagname[@attribute1='value' and @attribute2='value']

2.or
Syntax
------------
//tagname[@attribute1='value' or @attribute2='value']

==================================================
xpath with indexing/position()
==================================
Syntax
---------
(xpath)[index]

OR

(xpath)[position()=index]

=====================================================
xpath with methods
========================
1.text()
------------------
-Based on visible text of element we can identify that element
-text() never ignore white space

Syntax
----------
//tagname[text()='visibleText']

//a[text()='Logout']


2.normalize-space()
--------------------------
-Based on visible text of element we can identify that element
-normalize-space() ignore white space like trim()

Syntax
===========
//tagname[normalize-space()='visibletext']

3.contains()
--------------
class="oxd-icon bi-caret-down-fill oxd-userdropdown-icon"
         c1          c2                  c3

Syntax:
---------------
//tagname[contains(@attribute,'partial value')]

OR

//tagname[contains(text(),'partial text')]

example
------------
//i[contains(@class,'oxd-userdropdown-icon')]


4.starts-with()
--------------------
-Based on prefix value we can identify element
Syntax:
---------------
//tagname[starts-with(@attribute,'prefix value')]

OR

//tagname[starts-with(text(),'prefix text')]
